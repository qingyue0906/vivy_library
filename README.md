# Vivy Library

A ~~cross-platform~~ Flutter desktop app for managing a local media/content library. It scans file-system directories, reads per-folder `info.json` metadata, and presents everything in a VS Code-inspired three-panel UI with card-grid browsing, rich editing, and extensible scripting support.

## Features

- **File-system library scanner** — Recursively discovers folders containing `info.json`, builds a category tree
- **Three-panel layout** — Category tree (left), card grid (center), detail panel (right), all resizable
- **Responsive card grid** — Thumbnail previews, GIF support (3 playback modes), adjustable aspect ratio
- **Rich metadata editing** — Title, description, creator, type, rating, classes, tags, goto links
- **Batch edit** — Multi-select items and folders for bulk operations
- **Context menus** — Right-click items, folders, or files for edit, locate, open, rename, properties
- **Class chip filters** — Filter grid content by class tags with live item/folder counts
- **Python script engine** — Import `.py` scripts and run them on selected paths via context menus
  - Three modes: result dialog, terminal, silent
  - Per-script enable toggle, descriptions parsed from docstring
- **Full-text search** — Search by title, description, creator, tags, classes
- **Multi-language** — Chinese (Simplified), Chinese (Traditional), English, Japanese
- **Custom background** — Set a wallpaper with per-panel opacity
- **Data export/import** — `.zip` archive containing settings, library roots, and scripts
- **Compact mode** — Global UI scale slider (0.5×–2.0×)
- **VS Code dark theme** — Carefully tuned dark palette matching VS Code's aesthetics
- **Window state persistence** — Remembers position, size, and layout across restarts

## Info.json Format

Each folder inside a library root can contain an `info.json` file that describes its contents:

```json
{
    "title": "My Item",
    "description": "A description of this item",
    "creator": "Author Name",
    "type": "video",
    "contentrating": "G",
    "rating": 5,
    "class": ["Collection", "Favorites"],
    "tags": ["tag1", "tag2"],
    "goto": [
        { "uuid": "xxx-xxx-xxx" },
        { "path": "/full/path/to/related/folder" }
    ]
}
```

The library scanner automatically builds the category tree from your folder structure, and the info fields become searchable, sortable, and filterable metadata in the app.

## Project Structure

```
lib/
├── main.dart                   # App entry & theme setup
├── models/                     # Data models (LibraryItem, CategoryNode, etc.)
├── services/                   # Business logic & persistence
├── providers/                  # Central state management
├── widgets/                    # UI components
└── utils/                      # Utilities
```

User data is stored under `%APPDATA%/vivy_library/` (auto-migrated from `shared_preferences` on first launch):

```
%APPDATA%/vivy_library/
├── settings.json              # All app settings
├── scripts.json               # Script metadata
└── scripts/                   # Imported Python scripts
```
